# BookShelf AI Rules

## Project Rules
- Backend code lives in apps/api.
- Frontend code lives in apps/web.
- Shared types live in packages/shared.
- Do not add a database. Use JSON file storage only.
- Follow existing Express + TypeScript patterns.
- API errors should return `{ error: "message" }`.

## API Rules
- DELETE success must return 204.
- Missing book must return 404 with `{ error: "Book not found" }`.
- Search should support title, author, and genre.
- Do not change existing API response shapes without asking.

## Frontend Rules
- Use React + TypeScript.
- Use Tailwind CSS for styling.
- Mobile-first responsive layout.
- Book grid: 1 column mobile, 2 tablet, 3 desktop.

## What NOT to do
- Do not rewrite the whole project for a small change.
- Do not introduce MongoDB/PostgreSQL.
- Do not remove existing endpoints.
- Do not hardcode test data if API is available.
